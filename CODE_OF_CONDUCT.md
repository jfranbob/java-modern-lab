# Code of Conduct

## Our Commitment

Java Modern Lab is committed to providing a welcoming, respectful, and inclusive environment for everyone, regardless of experience level, background, identity, or location.

We believe that great engineering is built through collaboration, curiosity, and respectful discussion.

---

# Our Values

Every interaction within this project should reflect the following values:

* Respect
* Kindness
* Professionalism
* Curiosity
* Humility
* Continuous learning

Disagreements are expected.

Disrespect is not.

---

# Expected Behavior

Contributors are encouraged to:

* Be respectful and constructive.
* Explain technical opinions with evidence.
* Accept feedback professionally.
* Ask questions openly.
* Help others learn.
* Assume good intentions.

Engineering discussions should focus on ideas, never on people.

---

# Unacceptable Behavior

The following behaviors are not tolerated:

* Harassment
* Personal attacks
* Discrimination
* Trolling
* Intimidation
* Offensive language
* Deliberately spreading misinformation
* Repeatedly ignoring project guidelines

Technical disagreements should never become personal conflicts.

---

# Engineering Discussions

This project encourages healthy technical debate.

Different solutions often exist for the same problem.

When discussing architecture or implementation:

* Support arguments with evidence.
* Prefer measurements over assumptions.
* Prefer benchmarks over opinions.
* Prefer official documentation over speculation.

The best engineering decision is often context-dependent.

---

# AI Usage

AI-generated content is welcome.

However:

* Contributors remain responsible for everything they submit.
* AI-generated code must be reviewed.
* AI-generated explanations should be verified.
* Understanding is required before submission.

Transparency and technical accuracy are more important than speed.

---

# Enforcement

Project maintainers may edit, reject, or remove contributions that do not follow these guidelines.

Repeated violations may result in restricted participation.

These decisions are made to protect the quality of the project and the community.

---

# Final Thoughts

Java Modern Lab is more than a code repository.

It is a place for engineers who enjoy learning, experimenting, measuring, and sharing knowledge.

Let's build something that future developers can trust.
